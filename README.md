# 🚀 Git & DevOps Challenges - Roadmap de Aprendizaje

Este repositorio contiene la resolución práctica de una serie de desafíos técnicos de Git, enfocados en un flujo de trabajo profesional de DevOps. El entorno de trabajo se configuró utilizando **Vagrant** con una máquina virtual **Ubuntu**.

## 🛠️ Tecnologías y Herramientas
* **OS:** Ubuntu (vía Vagrant/Oracle VM)
* **Terminal:** PowerShell & Bash
* **Versionado:** Git & GitHub (Autenticación por SSH)
* **Editor:** Nano

## 📝 Desafíos Completados

### 1. Configuración e Infraestructura
- [x] Configuración de llaves **SSH** para autenticación segura.
- [x] Gestión de múltiples ramas y resolución de errores de configuración de email.

### 2. Gestión Avanzada de Commits
- [x] **Git Reset & Revert:** Aprendizaje de técnicas para deshacer cambios según la necesidad del historial.
- [x] **Git Amend:** Corrección de mensajes y contenido del último commit realizado.

### 3. Flujos de Trabajo y Colaboración
- [x] **Git Rebase:** Reorganización del historial para mantener una línea de tiempo limpia y legible.
- [x] **Resolución de Conflictos:** Gestión manual de colisiones de código (`<<<<<<< HEAD`) utilizando el editor Nano.
- [x] **Git Stash:** Almacenamiento temporal de cambios pendientes para alternar entre tareas sin perder progreso.

### 4. Versionado Profesional
- [x] **Git Tags:** Implementación de etiquetas de versión (`v1.0`) para marcar hitos importantes del proyecto.
